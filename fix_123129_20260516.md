# Fusion 360 スケッチツール 開発記録

## 技術スタック

純粋な **Vanilla JS + SVG** のみ。フレームワーク・ライブラリ一切不使用。単一HTMLファイルに全機能を封入。

---

## フェーズ 1 — 基盤構築

### ノード・エッジ描画ツール（初期版）

**入力仕様:** JSONファイル（`fusion_kousoku_vibe_v4.6`）を構造解析。

```json
{ "id": "N1778732776872", "content": "Reference：fusion 360 の拘束",
  "parentId": null, "createdAt": "13:26:16" }
```

`parentId` の有無でルートノードと子ノードを判別し、BFSによる自動レイアウトを実装。点と線のみで構成されるグラフ描画エンジンを構築。

**技術的特異点①: SVG 座標系の統一**

```js
function toW(sx,sy){ return {x:(sx-VX)/VS, y:(sy-VY)/VS} }
function toS(wx,wy){ return {x:wx*VS+VX,   y:wy*VS+VY}   }
```

ビュー変換（平行移動＋スケール）をすべてこの2関数に集約。SVG内の全レイヤーに同一 `transform` を適用することで、ズーム・パン時の再計算をゼロに抑制。

## フェーズ 2 — Fusion 360 UI の完全再現

### 外観・レイアウト

- Autodesk chrome風タイトルバー（#111背景）
- リボンUI（タブ切替 + グループ分割 + 最小幅44pxボタン）
- 左サイドバー（縦アイコン列）
- 右プロパティパネル（タブ切替）
- ボトムバー（ステータス表示）
- グリッド: マイナー20px / メジャー100px の2層パターン
- 原点軸: X=赤(#b03030) Y=緑(#308050)

**技術的特異点②: SVG `<pattern>` の入れ子**

```xml
<pattern id="gm" width="20" height="20">...</pattern>  <!-- 細グリッド -->
<pattern id="gM" width="100" height="100">
  <rect fill="url(#gm)"/>   <!-- 細グリッドを内包 -->
  <path .../>                <!-- 太線を重ねる -->
</pattern>
```

1回の `<rect>` 描画で2階層グリッドを実現。ズーム変換は親 `<g>` の `transform` のみで制御するため、グリッドも自動追従。

### 線分クリック判定の解決

**問題:** `stroke-width` をズームスケールで縮小すると実クリック領域が極細になりクリック不能になる。

**解法:** 見た目の線（細い）と透明なヒットエリア（太い・固定px）を分離。

```js
// 見た目
sa(vis, {'stroke-width': 1.5/VS, 'pointer-events':'none'});
// ヒットエリア — ズームに関係なく常に14px相当
sa(hit, {'stroke-width': 14/VS, stroke:'transparent', cursor:'pointer'});
```

## フェーズ 3 — 拘束エンジン実装

### JSON仕様からの拘束8種 + 結線2種

| 種別 | 実装内容 |
|------|---------|
| 水平垂直 | `|dx|≥|dy|` で水平/垂直を自動判定し端点を投影 |
| 一致 | 2点を同一座標へ。固定点がある場合は逆方向へ移動 |
| 等しい | 基準線長を `scaleSeg()` で従属線にコピー |
| 直行 | 基準線の法線方向ベクトルを従属線に適用 |
| 並行 | 基準線の方向ベクトルを従属線に適用（後述の修正あり）|
| 中点 | 線分端点の平均座標に点を投影 |
| 同一線上 | 直線へのベクトル射影（tクランプなし=延長線対応）|
| 固定 | `pt.fixed=true` でドラッグ・拘束演算の対象外化 |
| 寸法指定 | 数値入力→`scaleSeg()`で長さ強制 |
| 座標スナップ | 20pxグリッドへの丸め + 近傍点優先スナップ |

**技術的特異点③: ベクトル射影による同一線上拘束**

```js
// 直線BC上への点Aの射影
const t = ((pt.x-ax)*dx + (pt.y-ay)*dy) / l2;
pt.x = ax + t*dx;
pt.y = ay + t*dy;
// tをクランプしない → 線分の延長線上も含む直線全体に拘束
```

Fusion 360の "Coincident on Line" の挙動を忠実再現。線分端点を超えた延長方向への射影も有効。

## フェーズ 4 — 選択システムの再設計

### 問題: 複数選択が機能しない

**根本原因2つ:**
1. Shiftクリック時に `startDimOnLine()` が毎回呼ばれ状態リセット
2. 線分のクリックイベントが `pointer-events:none` 化されていた

**修正:** ヒットエリア分離（フェーズ2参照）＋選択ロジックの純粋化。

### 順序付き選択（親子モデルの基盤）

```js
selIds[0] = ①親（基準・動かない側）→ オレンジ (#ff9900)
selIds[1] = ②子（従う・動く側）  → シアン  (#4db8ff)
```

`selIds.length < 2` を上限として `push` することで自然に2要素制限。

ゴースト描画: 白(#ffffff) 不透明度0.3の破線で「適用後の姿」を半透明表示。移動軌跡も矢印線で可視化。

## フェーズ 7 — 先行拘束維持システム

**技術的特異点⑤: 拘束の連鎖再適用**

拘束は適用順でスタックに積まれ、点のドラッグや新規拘束適用のたびに **全拘束を登録順に再適用** する。

```js
function enforceAllConstraints(){
  for(const con of cons){   // 登録順を厳守
    enforceOne(con);
  }
}
```

これにより「水平垂直を適用した線に等しい拘束を追加」しても、水平垂直が維持されたまま長さが揃う。先行拘束が後続に破壊されない。

`hvDir` フィールドを拘束オブジェクトに記録し再適用時に水平/垂直の向きを保持:

```js
con = mkCon('水平垂直', s.id, [s.p2], [...], {hvDir: isH?'H':'V'});
```

## フェーズ 8 — 寸法拘束のUX改善

### 2モード設計

| モード | トリガー | 動作 |
|--------|---------|------|
| **選択ツール** | 線分クリック | 選択のみ（寸法パネルは開かない）|
| **選択ツール** | 線分を右クリック | コンテキストメニュー → 寸法編集 |
| **寸法ツール(D)** | 線分クリック | 即寸法入力フィールドを展開 |

### キャンバス内インライン入力

**技術的特異点⑥: SVG座標系上に浮遊するDOM input**

```js
// 線分の法線方向オフセット位置を計算
const nx=-dy/len, ny=dx/len, off=30/VS;
const sc = toS(mx+nx*off, my+ny*off);

// DOM inputをその位置に絶対配置
inp.style.left = (sc.x-38)+'px';
inp.style.top  = (sc.y-15)+'px';
```

ズームホイール時は `transform` の変化量から位置を再計算し、inputが寸法線に追従し続ける。

寸法ラベル（SVG `<text>`）も `pointer-events:visibleFill` でクリック可能にし、既存寸法の再編集をサポート。

### 右クリックコンテキストメニュー

```
↔ 寸法を編集      ← 線分のみ表示
─────────────
⊢ 水平垂直
⚓ 固定
─────────────
✕ 削除
```

画面端対策: `Math.min(e.clientX, window.innerWidth-180)` で右端からはみ出さないよう制御。

## フェーズ 9 — 並行拘束の方向保持修正

**問題:** 斜めの線分に並行拘束を適用すると、元の向きと逆方向に回転し交差するケースが発生。

**根本原因:** 方向ベクトルをそのまま適用すると、内積が負（逆向き）の場合に反転してしまう。

**技術的特異点⑦: 内積による向き判定**

```js
const bdx=p2b.x-p1b.x, bdy=p2b.y-p1b.y;
const dot = adx*bdx + ady*bdy;   // 親・子の方向ベクトルの内積
const sign = dot >= 0 ? 1 : -1;  // 同向き=+1、逆向き=-1

// sign を乗算することで「原型に近い向き」を選択
p2b.x = p1b.x + (adx/al)*lb*sign;
p2b.y = p1b.y + (ady/al)*lb*sign;
```

`enforceOne`・`cPara`・`computeGhost` の3箇所すべてに同一ロジックを適用。

## フェーズ 11 — Undo/Redo

**技術的特異点⑧: スナップショット方式の履歴管理**

```js
const UNDO_MAX = 50;
let undoStack = [];
let redoStack = [];

function snapshot(){
  undoStack.push({
    pts:  pts.map(p=>({...p})),
    segs: segs.map(s=>({...s, conTypes:[...(s.conTypes||[])]})),
    cons: cons.map(c=>({...c, children:[...c.children], targets:[...c.targets]})),
    uid:  _id,
  });
  if(undoStack.length > UNDO_MAX) undoStack.shift();
  redoStack = [];   // 新操作でRedoは消滅
}
```

**スナップショットタイミング（全8箇所）:**

| タイミング | 理由 |
|-----------|------|
| `addPt()` 呼び出し前 | 点追加を取り消せる |
| `addSeg()` 呼び出し前 | 線分追加を取り消せる |
| ドラッグ**開始時** | 移動前の座標を保存（終了後では遅い）|
| `execApplyC()` 前 | 拘束適用を取り消せる |
| `commitDim()` 前 | 寸法確定を取り消せる |
| `deleteSelected()` 前 | 削除を取り消せる |
| `clearAll()` 前 | 全消去を取り消せる |
| `restoreData()` — 読込時 | *(読込はデータ全置換のためundoStackはリセット)* |

**キーバインド:**
```
Ctrl+Z          → Undo
Ctrl+Y          → Redo
Ctrl+Shift+Z    → Redo（Mac慣習）
```

```

  { id, type,
```
